# DustMate — React 변환 작업계획서

> **원본**: `start/` (HTML + CSS + JS 퍼블리싱본)  
> **목표**: React + Vite + TypeScript + styled-components 프로젝트  
> **작성일**: 2026-06-15

---

## 1. 원본 소스 분석 요약

### 1-1. 화면 구성 (screen 단위)

| 화면 ID | 화면명 | 라우팅 전략 |
|---|---|---|
| `screen-login` | 로그인 | 앱 진입 (초기 상태) |
| `screen-home` | 홈 | 로그인 후 기본 화면 |
| `screen-diary` | 건강 다이어리 | 하단 탭 |
| `screen-group` | 친구·가족 그룹 | 하단 탭 |
| `screen-history` | 기록 히스토리 | 하단 탭 |

> 원본은 SPA 방식으로 `section.screen`의 `active` 클래스 토글로 화면을 전환함.  
> React 변환 시 **`react-router-dom`** 없이 **내부 상태(Context)로 화면 전환** 유지  
> (모바일 앱 UX 특성상 URL 기반 라우팅보다 상태 기반 스택 내비게이션이 적합)

### 1-2. 주요 기능 및 로직 (app.js 분석)

| 카테고리 | 함수 / 로직 | 설명 |
|---|---|---|
| API 연동 | `fetchDustAlarms()` | 에어코리아 API (PM10/PM2.5 경보 조회) |
| 위치 | `initLocation()`, `reverseGeocode()` | Geolocation + Nominatim 역지오코딩 |
| 라우팅 | `navigate(screenId)` | 화면 전환 + 하단 탭 활성화 |
| 인증 | `loginDemo()` | 데모 로그인 처리 |
| 홈 | `updateHomeScreen()`, `updateDustCard()` | 미세먼지 카드 + 다이어리 요약 갱신 |
| 경보 | `loadAlarmSection()`, `alarmRow()` | 경보 목록 렌더링 (내 지역/전국 분리) |
| 다이어리 | `initDiaryScreen()`, `saveDiary()` | 컨디션·증상·메모 저장 (localStorage) |
| 히스토리 | `initHistoryScreen()`, `renderCalendar()`, `renderHistoryList()` | 월간 캘린더 + 기록 목록 |
| 그룹 | `createGroup()`, `copyInviteLink()` | 그룹 생성·초대 |
| 모달 | `openModal()`, `closeModal()` | Bottom Sheet 모달 |
| 토스트 | `showToast()` | 피드백 토스트 알림 |
| 유틸 | `getTodayStr()`, `formatDate()`, `getAllDiaries()` 등 | 날짜 처리·스토리지 |

## 3. 기술 스택 및 패키지

| 항목 | 선택 | 비고 |
|---|---|---|
| 프레임워크 | React 18 + Vite | `create-vite` 사용 |

## 5. 상태 관리 설계

### AppContext 구조

```typescript
// src/context/AppContext.tsx
type Screen = 'login' | 'home' | 'diary' | 'group' | 'history';

interface AppState {
  currentScreen: Screen;
  isLoggedIn: boolean;
  location: LocationData | null;
  selectedCondition: ConditionType;
  selectedSymptoms: string[];
  calYear: number;
  calMonth: number;
  toastMessage: string;
}

type Action =
  | { type: 'NAVIGATE'; screen: Screen }
  | { type: 'LOGIN' }
  | { type: 'SET_LOCATION'; payload: LocationData }
  | { type: 'SET_CONDITION'; value: ConditionType }
  | { type: 'TOGGLE_SYMPTOM'; symptom: string }
  | { type: 'CHANGE_MONTH'; dir: 1 | -1 }
  | { type: 'SHOW_TOAST'; message: string }
  | { type: 'HIDE_TOAST' };
```

## 7. 화면 전환 애니메이션 구현 방식

원본 CSS의 `.screen`, `.screen.active`, `.screen.slide-out` 클래스 전환 로직을  
React에서는 **styled-components + 조건부 props**로 구현:

```typescript
const ScreenWrapper = styled.section<{ $isActive: boolean; $isSliding: boolean }>`
  position: absolute;
  inset: 0;
  display: flex;
  flex-direction: column;
  overflow: hidden;
  opacity: ${({ $isActive }) => ($isActive ? 1 : 0)};
  pointer-events: ${({ $isActive }) => ($isActive ? 'all' : 'none')};
  transform: ${({ $isActive, $isSliding }) =>
    $isSliding ? 'translateX(-30px)' : $isActive ? 'translateX(0)' : 'translateX(30px)'};
  transition: opacity 0.35s ease, transform 0.35s ease;
`;
```

> styled-components v6에서는 DOM에 전달되지 않아야 할 props에 `$` prefix를 사용합니다.

## 8. 주의 사항

> **원본 보존**: `start/` 폴더는 참고용으로 절대 수정하지 않습니다.

> **API KEY 관리**: 에어코리아 API KEY는 `src/constants/index.ts`에 보관하되,  
> 추후 `.env` 파일로 이전 (`VITE_AIR_KOREA_API_KEY`)을 권장합니다.

> **로그인**: 현재 데모(loginDemo) 방식이므로 실제 인증 연동 시 별도 작업이 필요합니다.

> **성능**: `useDiaryStorage` 훅에 `useCallback` / `useMemo` 적용으로  
> 불필요한 리렌더링을 방지하세요.
